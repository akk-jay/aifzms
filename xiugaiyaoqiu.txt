### 三、功能实现要求（核心）

#### 1. 系统设置页：DeepSeek API 配置
- 实现表单校验：所有带 `*` 的字段为必填项，提交前校验非空。
- 实现「保存配置」功能：
  - 将 API 密钥、URL、模型名、最大 Token 数保存到本地存储（`localStorage` 或 `electron-store`）。
  - 保存成功后显示「配置状态：✅ 已配置」和「最后更新：时间戳」。
  - 页面刷新后，自动读取已保存的配置并回填到表单。
- 实现「隐藏/显示 API 密钥」切换按钮。

#### 2. 首页：上传简历功能
- 实现简历上传功能：
  - 支持 `.pdf` / `.doc` / `.docx` 格式文件。
  - 文件上传后保存到本地（可以存在 `localStorage` 或项目指定目录）。
  - 上传成功后，提示文字从「你并未上传简历，AI 将无法基于简历回答，去上传」变为「已上传简历：文件名」，并显示文件名。
  - 支持删除/重新上传简历。

#### 3. 首页：创建问答库功能
- 点击「创建问答库」按钮，弹出弹窗或跳转到问答库管理页面。
- 实现基础的问答库创建功能：
  - 支持添加问题和答案（文本输入框）。
  - 支持保存、查看、编辑已创建的问答库。
  - 问答库数据保存到本地存储。

#### 4. 通用功能要求
- 所有表单提交都要有加载状态（按钮禁用 + loading 动画）。
- 保存/上传成功/失败要有 Toast 提示。
- 数据要持久化：刷新页面后，所有配置、上传的简历、问答库数据都要保留。
- 代码要结构清晰，使用 React Hooks 管理状态，表单使用受控组件。
