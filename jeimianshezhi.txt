一、整体产品定位
这是一款 本地部署的桌面级 AI 面试辅助工具 Web 应用，核心流程是：
用户在 Web 端完成配置（API 密钥、面试场景、简历 / 题库）
点击「前往面试」后，启动一个 Windows 桌面半透明悬浮窗口
窗口实时通过语音转文字（科大讯飞）+ AI 大模型（DeepSeek），获取面试官问题并生成文字回答，实现面试辅助。
二、页面 1：新建面试配置页（Web 端）
1. 页面结构
左侧导航栏（固定）
用户中心（含「首页」入口）
面试管理（展开状态，当前选中「新的面试」，含「面试记录」入口）
帮助、系统设置入口
右侧主内容区：线性配置流程，分三大模块，底部为提交按钮。
2. 模块细节
模块 1：面试基础信息
标题：请选择面试地区和岗位
配置项：
地区：下拉选择框（当前：简体中文），控制语音识别语言
岗位：下拉选择框（当前：前端），用于匹配 AI 面试场景与题库
补充入口：没有想要的岗位？联系小助手一键添加（支持用户自定义岗位）
模块 2：技术定制
标题：技术定制
配置项：
编程语言（仅针对笔试功能）：下拉选择框（当前：打字稿 / TypeScript），影响笔试题库与代码生成
是否使用自定义问答库：下拉选择框（是 / 否，当前：否）
状态提示：当前问答库为空 + 蓝色「创建问答库」按钮（用于用户上传 / 管理自定义题库）
模块 3：模式选择与提交
提示信息栏：浅蓝色背景提示框，文字为 你并未上传简历，AI将无法基于简历回答，去上传，「去上传」为跳转链接
提交按钮：底部居中蓝色主按钮 前往面试，点击后启动桌面悬浮窗口，进入面试模式。
三、页面 2：系统设置页 - 科大讯飞语音配置（Web 端）
1. 页面结构
左侧导航栏：同页面 1，当前选中「系统设置」
右侧主内容区：
标题：应用设置，副标题：配置API密钥和相关参数
顶部双标签页：「科大讯飞语音」（当前选中）/「DeepSeek AI」

表单字段（均为必填，带 * 标记）：

操作按钮：
蓝色主按钮 保存配置
灰色次按钮 清除配置
配置状态反馈：
状态文本：配置状态：✅ 已配置
时间戳：最后更新：2026/5/30 20:21:50
四、页面 3：系统设置页 - DeepSeek AI 配置（Web 端）
1. 页面结构
左侧导航栏：同前两页，当前选中「系统设置」
右侧主内容区：标签页切换为「DeepSeek AI」，展示大模型配置表单
2. 表单细节（DeepSeek API 配置）
表单字段（均为必填，带 * 标记）：
API 密钥：密码输入框，带「显示 / 隐藏」切换图标
API 基础 URL：文本输入框（默认值：https://api.deepseek.com）
模型名称：文本输入框（默认值：deepseek-chat）
最大 Token 数：数字输入框（默认值：2000）
温度（Temperature）：数字输入框（默认值：0.7）
操作按钮：底部蓝色主按钮 保存配置
五、面试模式：桌面悬浮窗口（核心功能）
1. 窗口基础特性
运行环境：Windows 桌面
外观：半透明黑色背景，无边框、极简纯净风格
行为：始终置顶、可拖动、可调节大小，不遮挡视频会议窗口
核心功能：实时语音转文字 + AI 回答生成
2. 工作流程
窗口启动后，持续监听麦克风输入
通过科大讯飞 API，将面试官的语音实时转文字
调用 DeepSeek API，结合用户配置（岗位、简历、题库）生成回答
回答内容实时显示在半透明窗口中，用户可直接阅读，无需切换界面
3. UI 设计要求
背景：半透明黑色（低透明度，不遮挡视频画面）
文字：白色 / 浅灰色，清晰易读，无多余装饰
交互：无复杂控件，仅显示问题和 AI 回答，可滚动查看历史内容
性能：轻量化，后台运行不占用过多系统资源
六、整体 UI 与交互规范
Web 端风格：极简白色背景，浅蓝色主色调，圆角卡片式表单设计，层级清晰
表单状态：必填项用 * 标记，API 密钥支持明文 / 密文切换，保存后显示配置状态与更新时间
提示反馈：配置错误 / 为空时，返回明确错误提示；配置成功时，显示「已配置」状态
跳转逻辑：「去上传」「创建问答库」「前往面试」等按钮均有明确跳转 / 启动逻辑
七、技术要点说明（给 Claude 的开发提示）
Web 端：用 React/Vue 实现，路由包含 index、new-interview、settings、records 等页面
桌面悬浮窗口：用 Electron/PyQt 实现，可通过 Web 端 API 控制启动 / 关闭，接收配置参数
API 调用：Web 端保存配置，桌面窗口读取配置，分别调用科大讯飞和 DeepSeek API
数据存储：配置信息（API 密钥、面试设置）保存在本地，支持多用户独立配置
